VPN через SSH: настройка сети с помощью systemd-networkd

С чего начать? Начну с самого начала...
Весна 2026 года. Россия. Москва. Вокруг сплошные блокировки. Все, что не популярно у программистов Яндекса, все блокируется. Какой выход? Ну, VPN... Может быть. Но и их уже дововльно просто блокируют. С осени прошлого года есть в наличии недорогой VPS. Вот его и решил использовать.
Какие цели стоят:
  • Доступ ко всем ресурсам
  • Доступ с компа и телефона
Пока стоял KDE - все было просто. Запускал ssh -D, в сетевых настройках proxy указывал на локальный порт и все работало. Но... Стоило сменить рабочий стол на lxde и все. Сразу скажу, что переменные прокси не работают так, как хотелось. Крутил и так, и сяк... Выход нашел один. Туннель до VPS + перенаправление трафика в этот туннель. Почему чистый systemd? Да устал тянуть весь хлам в систему "чтобы было проще". А по сути ни разу не проще. И так... Начнем!
Первым-наперво хочу заметить, что этот демон (systemd-networkd) по умолчанию выключен. Во всяком случае у меня было так. Прежде, чем включать, его нужно настроить. Затем нужно отключить все утилиты, которые работают с настройками сети. Иначе может получиться проблема, когда они вместе начнут менять настройки. У меня был только network-manager (nmcli). Сначала отключил, а потом и удалил. Почему сначала настройка? nmcli, например, когда останавливается, то после перезапуска системы сети не будет. Это я проверил. ))))
Данный демон забирает настройки, насколько я понял, из нескольких мест. Но я работал с каталогами:
/etc/systemd/network/
Данный каталог у меня в системы был пустой. Ну, давайте начнем.
Первым делом смотрим, какие интерфейсы уже есть в системе:
ip a
1: lo: mtu 65536 qdisc noqueue state UNKNOWN group default qlen 1000
link/loopback 00:00:00:00:00:00 brd 00:00:00:00:00:00 inet 127.0.0.1/8 scope host lo valid_lft forever preferred_lft forever 
2: enp3s0f0: mtu 1500 qdisc fq_codel state UP group default qlen 1000
link/ether c0:3f:d5:ea:a1:83 brd ff:ff:ff:ff:ff:ff
inet 192.168.0.253/24 brd 192.168.0.255 scope global enp3s0f0 valid_lft forever preferred_lft forever
inet6 fe80::c23f:d5ff:feea:a183/64 scope link valid_lft forever preferred_lft forever
И так. Вижу два соединения:
1: lo
2: enp3s0f0
lo — это локальная петля, а вот enp3s0f0 — это сетевая карта. Как видим здесь уже есть адреса. Но когда я снес nmcli, то этот интерфейс не был настроен и соответственно сеть не работала.
Идем в /etc/systemd/network/:
cd /etc/systemd/network/
Тут создаем файл с понятным именем и постфиксом .network. Содержимое моего файла, при котором сеть заработала:
[Match]
Name=enp3s0f0
[Network]
Address=192.168.0.253/24
Gateway=192.168.0.1
[Route]
Destination=192.168.0.0/24
Gateway=192.168.0.1
[Route]
Destination=0.0.0.0/0
Gateway=192.168.0.1
Теперь подробнее о наполнении данного файла. В квадратных скобках буду указывать секцию, к которой относится параметр.
[Match] Name — это имя интерфейса, к которому будут применяться настройки. Если переименовать имя сетевой карты в udev, то скорее всего, после этого действия, карта не будет настроена.
[Network] Address — здесь В РУЧНУЮ задаем адрес нашей карты.
[Network] Gateway — адрес шлюза, который связывает машины в нашей локальной сети
[Route #1] Destination — это локальный маршрут. То есть, когда пытаемся обратиться к устройству в локальной сети, данные пойдут через Gateway.
[Route #1] Gateway — это адрес шлюза через который будет идти связь с локальной сетью. Говорят, что данный маршрут не нужен, так как в [Network] все есть, но… Пусть будет. )))
[Route #2] Destination — указываем 0.0.0.0/0. Это будет маршрут по умолчанию. То есть, когда мы обратимся к адресу, отличному от нашей локальной сети, весь трафик пойдет на указанный шлюз. А он в свою очередь направит трафик на другой сети, который он знает.
[Route #2] Gateway — адрес шлюза для маршрута по умолчанию (маршрут default).
Ну, по сути все. Сеть настроена. Для файла .network есть еще множество параметров, но основные настройки уже есть.
Как применить настройки к сети?
Сначала включаем службу, то есть перегружаем демона systemd:
sudo systemctl daemon-reload
Теперь добавляем сетевой демон в автозагрузку:
sudo systemctl enable systemd-networkd.service
Можно сразу запустить его, добавив - - now или отдельной командой:
sudo systemctl start systemd-networkd.service
Если внести изменения в файл .network, то применить изменения можно с помощью:
sudo systemctl reload systemd-networkd.service
Не проходят изменения? Загляни в journalctl или перегрузи демона systemd, выключи и включи службу systemd-networkd с помощью:
sudo systemctl stop systemd-network.service
sudo systemctl start systemd-network.service
Первый этап окончен.
