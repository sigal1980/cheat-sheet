Как всегда, в systemd-networkd для настройки используются файлы. Для создания виртуальных сетевых устройств используем файл .netdev. Эти файлы называем осмысленно и располагаем в /etc/systemd/network/.
Ниже пример этого файл с пояснениями:
[NetDev]
Name=tun5
Kind=tun
[Tun]
User=user_name
[NetDev] Name=tun5 - здесь указываем имя нашего tun-устройства. Я так понял, что можно оставить автовыбор (оставить пустым), но это имя будет использоваться в команде ssh. Поэтому даем ему имя.
[NetDev] Kind=tun - это семейство, к которому относится создаваемое устройство. Есть множество виртуальных устройств, но нам нужен tun.
[Tun] User=user_name - описывает кому разрешен доступ. Я вписал свой логин. root не обязателен.
Настроек еще множество, устройств тоже, но мне нужны только эти.
После сохранения файла не забываем дать понять systemd, что файл создан/обновлен:
sudo systemctl daemon-reload
И также перегружаем systemd-networkd:
sudo systemctl reload systemd-networkd.service
По сути для создания tun-интерфейса этого достаточно. Далле нужно сделать сетевые настройки этого устройства. Для этого используем еще один файл .network. Желательно дать им одинаковые имена, чтобы видеть их связь с одним устройством. Например ssh-tun.netdev и ssh-tun.network.
Пример файла .network:
[Match]
Name=tun5
[Address]
Address=192.168.1.2/24
Peer=192.168.1.1/24
[Network]
Description="VPN over SSH"
DHCP=no
Address=192.168.1.2/24
[Match] Name=tun5 - имя устройства, которое мы настраиваем
[Address] Address=192.168.1.2/24 - ip-адрес, который будет присвоен устройству
[Address] Peer=192.168.1.1/24 - ip-адрес, с которым будет связан туннель
[Network] Description="..." - описание нашего устройства
[Network] DHCP=no - выключаем получение адреса от dhcp-сервера
[Network] Address=192.168.1.2/24 - Адрес присвоенный устройству
Сохраняем. Перегружаем демоны. В итоге получаем:
